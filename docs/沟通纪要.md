## 二、节奏玩法（改动最多）

### 2.1 基础机制

- 屏幕中心 **Target 圆脸**（100px，简洁可爱风，眼睛约 1500ms 左右看）
- 外圈从屏幕外缩入，按颜色操作：
  - 🟡 **黄圈**：到边缘 **单击**
  - 🔵 **蓝圈**：到边缘 **长按约 3 秒**
  - 🔴 **红圈**：到边缘 **连击**
- 判定基于 **环与目标圆半径重合**，而非纯时间偏差
- 背景 emoji 玩具随 BPM 跳动；程序化 BGM（无需外部音频文件）

### 2.2 判定规则（多轮迭代）

- Perfect / Good / Miss 范围反复调整（出现过过严、全 Miss、全 Perfect 等情况）
- 最终方向：**浅黄边缘完全重合 = Perfect**，部分重叠 = Good，过早/过晚/未点/进内圈 = Miss
- 去掉辅助判定线（只说明位置，不画出来）
- 黄圈判定带、重叠带多次微调；曾用脚本模拟 Perfect/Good/Miss 概率

### 2.3 红圈参数（当前）

| 参数 | 值 |
|------|-----|
| 连点窗口 | **1.25 秒** |
| Perfect | **6 次** |
| Good | **4 次** |
| 引导文案 | 含「自己估算好时间哦」 |

### 2.4 节奏时长与出圈逻辑

- 阶段时长：早期 10s/15s → 正式关 **60～105s**（随关卡递增）
- 黄圈：**1～3 个随机批次**；正式关用权重 + 预算（约 60s 内 2 蓝 + 3 红）
- 红/蓝 **独占窗口**：独占期间其它圈不能抢中心；可预出下一圈但需留衔接空档
- 黄圈须 **全部点完** 才出红/蓝；蓝 Miss 直接下一波；红圈结束可补黄圈
- 修复过：缩放异常、点不了、误 Miss、卡顿、Score 卡住不缩圈等

### 2.5 界面与反馈

- **仅节奏模块** 竖屏 1080p，嵌入横屏游戏
- 竖屏居中，目标圆约占屏宽一半；**整圆**随当前缩圈变黄/蓝/红
- 飘字改到 **圆圈外上方**；Perfect 光环从 **外缘** 扩散
- 黄圈点击缩放；蓝圈长按缩放 82→92；去掉红圈下方秒数/次数 UI
- 两侧背景：从太素 → 加装饰 → 去多余光圈 → 背景色与中间统一

## 三、部署 / 塔防 / 道具

### 3.1 部署

- 6×6 棋盘，拖拽或 **点击选手牌 → 点空格放置**
- 同类型同阶相邻 **合成**；合成次数有限
- 手牌左上角，相同道具 **堆叠 + 数量角标**
- 长按/悬停 **Tooltip** 看属性；右下角显示 lv1～4
- 职业区分：近战/远程/护堡 不同边框形状
- **去掉部署倒计时**
- **Bug 已修**：点棋盘士兵再点空白会复制 → 棋盘层防重复 + 记录原格

### 3.2 塔防

- 怪物：emoji（👹🧟🐉🦂🐍🦁🦖🕷️👾🎃）
- 道具：音乐主题 emoji
  - 节拍拳：🪇🪘🥁🎯
  - 毒囊炮：🪈🎺🎷📢
  - 弧光术：🪉🪗🎹🎸
  - 岩盾仪：⭐🌟🌙☀️
- 核心 **修复**：音律币、满血/不足提示、高亮与可点击性多次修复
- 去掉守卫开始后棋盘/手牌 **蒙板**
- 怪物打塔 **受伤音效**
- 每关 **核心视觉 + BGM** 随章节变化（A2+B2 方案）
- 波次提示文字改深色；L08 去掉闪电

### 3.3 数值与平衡

- 节奏道具：按 **B/A/S/SS/SSS** 评级发放，控制总量（避免一次 40 个）
- 每关道具数、怪物 HP/伤害/移速、塔攻/攻速递进
- **战斗模拟器** 跑 A/S/SS/SSS 通过率
- **核心修复次数上限**：L01–03 限 1 次，L04–06 限 2 次，L07 限 3 次，L08 限 4 次
- 按关卡敌人组成调整掉落（如飞行关多给可打飞行道具）

## 四、关卡 / UI / 存档

### 4.1 关卡选择

- 列布局：**关卡名 → 评级 → 星级 → 时长**（总宽 400px）
- 评级 **B/A/S/SS/SSS**，SSS 炫酷字体
- 星星 **可用 | 累计** 在 **右上角**，与章节名平行
- 未解锁飘字下移，不挡「选择关卡」
- L08 解锁：按 **累计获得星级** 解锁

### 4.2 菜单与说明

- 曲谱编辑器 → **玩法说明** 弹窗
- **线上正式版隐藏**：战斗模拟器、Meta 升级（**localhost 保留**）
- 节奏结算「节奏结束」四行文字 **加大行间距**

### 4.3 胜利 / 星级 / 分数

- **胜利**：波次出完 + 场上无敌人
- **失败**：核心 HP ≤ 0
- 节奏结算显示 **节奏分**（非「准确度 100%」）
- 左上角 Score：当前节奏得分

### 4.4 存档

- IndexedDB：`rhythm-guard-save`，每浏览器一份
- 换设备/浏览器不同步；清存档可重看欢迎窗

## 五、新手引导

- 方案：**独立 tutorial 关 + 三阶段 Coach**
- 首次进入 **欢迎窗**（开始引导 / 稍后再说）
- 节奏：点「知道了」后 **才开 BGM 和缩圈**；弹板时 **暂停节奏**
- 引导关固定顺序：**黄红蓝 → 黄蓝红**，可循环；**正式关逻辑不动**
- 部署引导文案、层级、跳过按钮大小多次调整
- 教程：更多部署物、2 次合成、修复教练、通关满血修复

## 六、打包 / 发布 / 试玩

### 常用命令

```bash
npm run dev              # 开发（5173）
npm run deploy:pages     # 构建 HTML5 产物到 release/
npm run pack             # zip 试玩包 + 试玩说明.txt
npm run serve:release    # 本地静态服务（8080，占用则自动换端口）
git push                 # 推送后 GitHub Actions 自动部署
```

### 链接

| 用途 | 地址 |
|------|------|
| **线上试玩（推荐分享）** | https://naanjing0625-ship-it.github.io/rhythm-guard/ |
| **Git 仓库** | https://github.com/naanjing0625-ship-it/rhythm-guard |
| **本地试玩** | http://localhost:8080/ 或终端显示的端口 |

### 分发说明

- **HTTPS 链接**：最稳定，推荐发给朋友
- **zip / 双击 HTML**：可能因浏览器、`file://`、微信内置浏览器失败
- `npm run dev` 不能当长期在线服务；线上靠 GitHub Pages
- push 后 Actions 自动部署，一般无需手动操作（首次需在 Settings → Pages 选 GitHub Actions）
